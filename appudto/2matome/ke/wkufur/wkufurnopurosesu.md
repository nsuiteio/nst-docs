## 設定手順

### ① 画面左のパネルで、プロセスを設定するチームをクリック

![](<../../../../.gitbook/assets/Screen Shot 2022-02-04 at 17.55.26.png>)

### ② チーム画面で、「プロセス」タブを選択し、アクションボタンをクリック

![](<../../../../.gitbook/assets/Screen Shot 2022-02-04 at 17.55.26 2.png>)



### ③ アクションボタンのクリック後、表示されたポップアップで「設定」をクリック

![](<../../../../.gitbook/assets/Screen Shot 2022-02-04 at 17.55.32.png>)

### ④ 各種項目を設定して、「保存」ボタンをクリック

![](<../../../../.gitbook/assets/Screen Shot 2022-02-04 at 18.34.38.png>)

{% hint style="info" %}
**各種設定項目の説明**
{% endhint %}

| 設定項目            | 説明                                                                                                             |
| --------------- | -------------------------------------------------------------------------------------------------------------- |
| ①プロセス名          | プロセスの名前です。現時点では、1チームに1プロセスしか設定できませんが、今後、チームに複数のプロセスを設定できるようになる予定です。用途に応じた名前を設定することで、プロセスの用途がわかりやすくなります。        |
| ②「申請者自身を除く」チェック | チェックが入っている場合、申請者自身では、承認及び実行ができなくなります。承認者及び実行者が自身で申請を行なった場合、自身では承認/実行をできません。                                    |
| ③ステップ名（承認ステップ）  | <p>承認ステップの名前です。申請画面にステップ名が表示されます。<br>【例】<br>「経理部承認」、「部門長承認」、「役員承認」など、ステップに名前を付けることで、承認の目的をわかりやすくすることが出来ます。</p> |
| ④承認者            | 申請に対して、承認を行うユーザーです。承認者は、複数人設定可能です。設定した承認者のうち、誰か1人が承認すれば、承認が完了します。                                              |
| ⑤ステップ名（実行ステップ）  | 実行ステップの名前です。申請画面にステップ名が表示されます。                                                                                 |
| ⑥実行者            | 承認済みの申請に対して、N Walletで署名を行うユーザーです。設定した実行者のうち、誰か1人が実行すれば、実行が完了します。                                               |
