# nst-docs

n-suite のドキュメントを GitBook と連携して管理するためのリポジトリです。

## 目的

- GitBook に公開するドキュメントのソースを Git で管理する
- ドキュメント変更履歴をコミットとして追跡する
- GitBook 側の同期対象ブランチと連携し、Markdown ファイルを継続的に更新する

## 想定構成

```text
.
├── README.md
└── SUMMARY.md
```

`README.md` は GitBook のトップページとして扱います。目次やページ構成が必要になった場合は、`SUMMARY.md` を追加して管理します。

## 更新フロー

1. Markdown ファイルを編集する
2. 変更内容を確認する
3. コミットして GitBook と連携しているブランチへプッシュする
4. GitBook 側で同期結果を確認する

## 運用メモ

- 画像や添付ファイルを追加する場合は、参照先が GitBook 上でも解決できる相対パスにします。
- ページ追加時は、必要に応じて `SUMMARY.md` に導線を追加します。
- 公開前に表記ゆれ、リンク切れ、機密情報の混入がないことを確認します。
